  <action>
    在 repo 根目錄建立 index.html。文件宣告使用 lang="zh-TW" 及 UTF-8。

    HTML 結構必須包含以下元素，ID 與標籤不可變更：
    - id="num1"：第一個數字輸入框，type="number"，placeholder="第一個數字"（per UI-01）
    - id="num2"：第二個數字輸入框，type="number"，placeholder="第二個數字"（per UI-01）
    - 四顆按鈕，標籤依序為「加」「減」「乘」「除」，各自的 onclick 屬性呼叫 calculate('add')、calculate('subtract')、calculate('multiply')、calculate('divide')（per UI-02）
    - id="result"：結果顯示區塊（div 或 p 元素），初始文字可為空白（per UI-03）

    JavaScript 函式 calculate(op) 的完整邏輯（依序執行，不可改變順序）：

    步驟 1：讀取 document.getElementById('num1').value 和 document.getElementById('num2').value，分別以 parseFloat() 轉換為數值 a 和 b。

    步驟 2：若 a 值為 NaN，或 num1 的 value.trim() 為空字串，或 b 值為 NaN，或 num2 的 value.trim() 為空字串，則將 id=result 元素的 textContent 設為「錯誤：請輸入有效的數字」，並立即 return（per ERR-02）。

    步驟 3：若 op 為 'divide' 且 b === 0，則將 id=result 元素的 textContent 設為「錯誤：除數不可為零」，並立即 return（per ERR-01）。

    步驟 4：依 op 值計算結果：'add' 執行 a + b，'subtract' 執行 a - b，'multiply' 執行 a * b，'divide' 執行 a / b。結果存入變數 answer（per CALC-01~04）。

    步驟 5：將 id=result 元素的 textContent 設為 answer。

    安全強制要求（per 威脅模型 T-01-01 和 T-01-02）：
    - 結果賦值一律使用 result.textContent = answer，絕對禁止 innerHTML、insertAdjacentHTML、outerHTML 等任何 DOM 注入介面
    - 計算邏輯一律使用 JavaScript 原生算術運算子（+、-、*、/），絕對禁止 eval()、new Function()、setTimeout/setInterval 傳字串等動態執行路徑
    - input 的 type="number" 限制鍵盤輸入，但 JS 側仍必須做 parseFloat + isNaN 二次驗證（per T-01-03，防止 Infinity 或 NaN 字串出現在結果區塊）
  </action>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| 使用者鍵盤輸入 → JavaScript | 使用者輸入的字串由 parseFloat() 解析；type="number" 不足以阻擋所有非法值，JS 側需二次驗證 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | result 元素 DOM 賦值 | mitigate | 一律使用 result.textContent = answer，禁止 innerHTML / insertAdjacentHTML，防止 XSS 注入 |
| T-01-02 | Tampering | calculate() 計算邏輯 | mitigate | 禁止 eval() / new Function() / setTimeout(string)，一律用原生算術運算子（+, -, *, /） |
| T-01-03 | Information Disclosure | 計算結果洩出 Infinity / NaN | mitigate | 除以零和無效輸入在步驟 2~3 攔截，永遠不讓 Infinity 或 NaN 字串進入 result.textContent |
| T-01-04 | Elevation of Privilege | 全頁面 | accept | 純靜態 file:// 頁面，無網路請求、無 cookie、無 localStorage、無外部資源，整體 attack surface 極小 |
| T-01-SC | Tampering | 套件安裝（slopcheck） | accept | 此專案無 package manager、無 npm/pip/cargo 安裝，此威脅不適用 |
</threat_model>
